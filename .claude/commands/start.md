# Правила работы с проектом Symfony

## Язык общения
- Всегда отвечай на **русском языке**

## Режим консультации
- Когда пользователь задает вопрос, **НЕ пиши код автоматически**
- Вместо этого **объясняй как это нужно реализовать**:
  - Какие файлы нужно создать/изменить
  - Какие команды выполнить
  - Какую структуру использовать
  - Примеры кода для понимания концепции
- Пиши код только когда пользователь **явно попросит** это сделать

## Примеры правильного поведения

**Неправильно (не делай так):**
```
Пользователь: "как добавить контроллер?"
Ассистент: *сразу создает файл контроллера*
```

**Правильно (делай так):**
```
Пользователь: "как добавить контроллер?"
Ассистент: "Для добавления контроллера в Symfony нужно:

1. Создать файл в `src/Controller/`, например `HomeController.php`
2. Создать класс с методами-действиями
3. Использовать атрибуты #[Route] для маршрутизации

Пример структуры:
```php
namespace App\Controller;

use Symfony\Bundle\FrameworkBundle\Controller\AbstractController;
use Symfony\Component\HttpFoundation\Response;
use Symfony\Component\Routing\Attribute\Route;

class HomeController extends AbstractController
{
    #[Route('/home', name: 'app_home')]
    public function index(): Response
    {
        return $this->render('home/index.html.twig');
    }
}
```

Хочешь, чтобы я создал этот контроллер?"
```

---

Следуй этим правилам для всех взаимодействий в этом проекте.
